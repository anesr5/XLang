# XLang v0.1 Language Reference

**Canonical specification for the bootstrap compiler (`x` 0.1.0).**

This document describes **only** what the current compiler implements. For release notes see [v0.1 release notes](../releases/v0.1.md). For topic-split navigation see [docs/spec/](README.md).

If this document and an RFC disagree, **this document and the compiler win** for v0.1 behavior.

## Overview

### Source files

| Property | Value |
|----------|-------|
| Encoding | UTF-8 (BOM rejected) |
| Extension | `.x` |
| Line endings | LF or CRLF |

### Program shape

```text
[ module_decl ]
{ import_decl }
{ struct_decl | function_decl }
EOF
```

Every complete program defines **`i32 main()`** with no parameters. The returned `i32` is the process exit code when run via `x run`.

### Pipeline

```text
source (.x) → lex → parse → type check → LLVM IR → verify → (clang) → executable
```

There is **no C backend** and no intermediate C generation.

### Minimal program

```xlang
module main

i32 add(i32 a, i32 b) {
    return a + b;
}

i32 main() {
    i32 x = add(40, 2);
    return x;
}
```

## Lexical structure

### Whitespace

Space, tab, carriage return, and newline separate tokens and are otherwise ignored.

### Comments

| Form | Status |
|------|--------|
| `// …` | Supported — to end of line |
| `/* … */` | Supported — non-nesting |
| `/// …` | Skipped like a line comment |

### Identifiers

```ebnf
identifier = ( ASCII-letter | '_' ), { ASCII-letter | ASCII-digit | '_' } ;
```

ASCII only. Identifiers cannot spell a **reserved keyword** (see below).

### Keywords used in v0.1

| Keyword | Role |
|---------|------|
| `module` | Module declaration |
| `import` | Import declaration |
| `struct` | Struct declaration |
| `const` | Immutable local binding |
| `return` | Return statement |
| `if`, `else` | Conditional statement |
| `true`, `false` | Boolean literals |

### Reserved for future syntax

These tokenize as keywords but are **not** in the v0.1 grammar:

```text
enum  trait  match
for  while  loop  break  continue  defer
async  await  parallel  spawn  gpu
unsafe  pub  impl  where
static  type  sizeof  alignof
move  mut  as  in
```

`fn`, `let`, and `var` are **not keywords**. They are valid identifier spellings.

### Literals

| Literal | v0.1 status |
|---------|-------------|
| Decimal integer | Supported — must fit `i32` in expressions |
| `"string"` | Supported in frontend — codegen rejects `str` |
| `true` / `false` | Supported |
| Float (`3.14`) | Lexed only — rejected in expressions |
| Char (`'a'`) | Lexed only — rejected in expressions |

Integer special case: unary `-2147483648` is allowed as `i32::MIN`.

### Operators and punctuation

**In expressions:** `+ - * / %`, comparisons, `== !=`, `&& ||`, unary `- !`, parentheses.

**Lexed but not in expression grammar:** `& | ^ ~ << >> -> => . :: ? [ ]`

### Semicolons

Executable statements **require** `;`. Newlines do not terminate statements.

## Grammar

### Module and imports

```ebnf
module_decl = "module", identifier ;
import_decl = "import", identifier ;
```

Not semicolon-terminated. No semantic effect in v0.1.

### Struct declarations

```ebnf
struct_decl = "struct", identifier, "{", { field_decl }, "}" ;
field_decl  = type_name, identifier, ";" ;
```

Parsed and validated. Not usable in expressions, signatures (except as rejected named types in fields), or codegen.

### Functions

```ebnf
function_decl = type_name, identifier, "(", [ param_list ], ")", block ;
param_list    = param, { ",", param } ;
param         = type_name, identifier ;
block         = "{", { statement }, "}" ;
```

C-style: return type **before** name. No `fn` keyword. No `->` return suffix.

```xlang
i32 add(i32 a, i32 b) { return a + b; }
void tick() { return; }
i32 main() { return 0; }
```

### Type names

Built-in spellings: `i32`, `bool`, `str`, `void`.

Other identifiers are **named types** (struct names in field lists). Named types are **rejected in function signatures** by the type checker.

### Statements

```ebnf
statement       = binding_stmt | assignment_stmt | return_stmt | if_stmt | expr_stmt ;

binding_stmt    = [ "const" ], type_name, identifier, "=", expr, ";" ;
assignment_stmt = identifier, "=", expr, ";" ;
return_stmt     = "return", [ expr ], ";" ;
if_stmt         = "if", expr, block, [ "else", block ] ;
expr_stmt       = expr, ";" ;
```

| Form | Mutability |
|------|------------|
| `type name = expr;` | Mutable — may be reassigned |
| `const type name = expr;` | Immutable — assignment is an error |

Type annotation is **required** on every binding.

`if` is a **statement**, not an expression. No semicolon after `}`.

### Expressions

Standard precedence (lowest to highest):

1. `||` (short-circuit)
2. `&&` (short-circuit)
3. `==` `!=`
4. `<` `<=` `>` `>=`
5. `+` `-`
6. `*` `/` `%`
7. Unary `-` `!`
8. Primary: literals, identifiers, calls, parentheses

All binary operators are left-associative except `&&` and `||`.

## Types

| Type | Frontend | LLVM codegen |
|------|:--------:|:------------:|
| `i32` | yes | yes |
| `bool` | yes | yes |
| `void` | return only | yes |
| `str` | yes | **no** |
| Named (struct) | struct fields | **no** |

### `void` rules

| Context | Allowed |
|---------|:-------:|
| Function return type | yes |
| Parameter / local / expression / return value / argument | no |
| Expression statement (`void_fn();`) | yes |

### Type equality

No implicit conversions. Assignment, arguments, returns, and `==` / `!=` require **exact** type match.

## Declarations

### Functions

- Return type first, then name, then parameters (`type name`).
- Parameter names unique per function.
- Function names unique per program.
- Parameters are **mutable** in the body.
- Forward calls allowed.

### Locals

Introduced only by binding statements inside functions. No block-scoped redeclaration across the function's flat namespace (see Semantics).

### `main`

Must exist exactly as `i32 main()` with empty parameter list.

---

## Statements

### Return

- `return expr;` — expression type must match function return type.
- `return;` — only in `void` functions.

### Assignment

`name = expr;` — target must be a parameter or mutable local; types must match.

### If / else

Condition must be `bool`. Both branches are blocks. For definite return through `if`, **both** branches must definitely return.

### Expression statements

Void-returning calls may appear as statements: `tick();`

---

## Expressions

### Calls

`callee(arg, …)` — arity and argument types must match the callee signature.

### Integer division / remainder

Literal divisor `0` is rejected at compile time.

### Strings

String literals type as `str` in the frontend. LLVM lowering rejects programs that require `str` codegen.

## Semantics

### Scoping

| Name | Scope |
|------|-------|
| Function | Whole program |
| Parameter / local | Enclosing function |
| Struct | Declaration only |

No import or module resolution.

### Branch visibility

A local declared inside an `if` branch is **not visible** outside that branch for **use**, but its name **cannot be redeclared** later in the function.

### Definite return

Non-void functions must return on all paths. Top-level `if` with `else` counts only if both branches definitely return.

### Unreachable code

Statements after a definite `return` on the same path are errors.

### Diagnostics

Errors carry stable family codes: `E-LEX-*`, `E-PARSE-*`, `E-TYPE-*`, `E-BACKEND-*`, with source spans (line/column).

## Compilation

### CLI

| Command | Output |
|---------|--------|
| `x check file.x` | Success or diagnostic |
| `x emit-llvm file.x` | Verified IR on stdout |
| `x build file.x` | `build/main.ll`, native binary |
| `x run file.x` | Execute; exit code = return value |

### Backend

Direct lowering via Inkwell to LLVM 22. Locals use stack slots (`alloca` / `load` / `store`). `Module::verify()` runs before IR is emitted or written.

### Environment

- `XLANG_TARGET_TRIPLE` — optional default target triple
- `LLVM_HOME` — Windows LLVM install path when not in default location

## Limitations

### Language

| Not supported | Notes |
|---------------|-------|
| Type inference | Explicit type on every binding |
| Multi-file / linking | Single translation unit |
| Import / module semantics | Parsed only |
| Struct values | Declarations only |
| Loops, `match`, `defer` | Not in grammar |
| `if` expressions | Statement only |
| Generics, ownership, `unsafe` | Not in v0.1 |
| Standard library | None |

### Tooling

- No official formatter
- LSP is experimental (`LSP/`)
- Fixed output paths: `build/main.ll`, `build/main.exe`

### Validation gap

Programs using `str` may pass `check` and fail at `emit-llvm`. Always run the full pipeline for complete programs.
