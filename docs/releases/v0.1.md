# XLang v0.1 Release Notes

**Status:** MVP bootstrap release  
**Compiler crate:** `x` 0.1.0  
**Date:** 2026

XLang v0.1 is the first documented, test-backed language snapshot. It delivers a small C-style surface language, a deterministic frontend, and direct LLVM IR lowering suitable for local experimentation and editor integration.

---

## Highlights

| Area | v0.1 delivery |
|------|----------------|
| Language | Single-file programs with C-style functions and locals |
| Types (codegen) | `i32`, `bool`, `void` |
| Types (frontend) | `str`, named struct types in struct fields |
| Control flow | `if` / `else`, `return` |
| Toolchain | `check`, `emit-llvm`, `build`, `run` |
| Editor | Experimental LSP + VS Code extension in `LSP/` |
| Tests | 65+ compiler unit tests, LLVM IR snapshots |

## Language surface

### Syntax model

XLang v0.1 uses **C-style declarations** throughout:

```xlang
module main

i32 add(i32 a, i32 b) {
    return a + b;
}

i32 main() {
    i32 x = add(40, 2);
    return x;
}
```

- Functions: `return_type name(type param, …) { … }`
- Mutable locals: `type name = expr;`
- Immutable locals: `const type name = expr;`
- Entry point: `i32 main()` with no parameters

There is **no** `fn`, `let`, or `var` keyword. Those spellings are ordinary identifiers.

### Supported today

- Integer, boolean, and string literals
- Function calls (forward references allowed)
- Arithmetic, comparison, logical `&&` / `||` (short-circuit in LLVM)
- Assignments to parameters and mutable locals
- Optional `module` and `import` declarations (parsed, not resolved)
- `struct` declarations (parsed and validated, not usable in code yet)

### Not supported

See [Limitations](../spec/limitations.md) and [v0.1 language reference](../spec/v0.1-language-reference.md).

Notable gaps: multi-file linking, struct values, loops, `match`, generics, ownership, standard library, official formatter.

## Toolchain

### Prerequisites

- Rust (edition 2024)
- LLVM 22 development libraries
- `clang` on `PATH` for `build` and `run`

### Commands

```bash
cargo run --manifest-path compiler/Cargo.toml -- check  examples/v0.1/main.x
cargo run --manifest-path compiler/Cargo.toml -- emit-llvm examples/v0.1/main.x
cargo run --manifest-path compiler/Cargo.toml -- build  examples/v0.1/main.x
cargo run --manifest-path compiler/Cargo.toml -- run    examples/v0.1/main.x
```

| Command | Purpose |
|---------|---------|
| `check` | Lex, parse, type-check |
| `emit-llvm` | Produce verified LLVM IR on stdout |
| `build` | Write `build/main.ll`, link with `clang` |
| `run` | Build and execute; exit code = `main` return value |

**Target triple:** `--target <triple>` or `XLANG_TARGET_TRIPLE`.

## Known issues

| Issue | Workaround |
|-------|------------|
| `str` passes `check` but fails codegen | Run `emit-llvm` or `run` for full validation |
| UTF-8 BOM rejected | Save files without BOM |
| Float / char literals lex but do not parse | Avoid in v0.1 programs |
| `check` vs backend gap | Treat `run` as the final gate for complete programs |
