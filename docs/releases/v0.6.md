# XLang v0.6 — Enum types in function signatures

**Status:** shipped in bootstrap compiler  
**RFCs:** [RFC-0038](../rfcs/RFC-0038-v0-6-roadmap-and-scope.md) through [RFC-0042](../rfcs/RFC-0042-v0-6-diagnostics.md)

---

## Summary

v0.6 lets **enum types appear in function parameters and return types**, including **cross-module `pub enum`** usage. The RFC-0035 `divide` / `ResultI32` pattern is now executable end-to-end.

---

## Language changes

| Feature | v0.5 | v0.6 |
|---------|------|------|
| Enum locals + `match` | yes | yes |
| Enum fn params / returns | — | yes |
| `return Ok(x)` / `return Err(n)` | — | yes |
| `match f()` on call result | — | yes |
| `pub enum` cross-module | parsed | usable in signatures |
| Struct in fn signatures | — | still rejected |

---

## LLVM ABI

Enum parameters and returns use the v0.5 tagged struct **`{ i32 tag, i32 payload }`** passed **by value**.

---

## Examples

See [examples/v0.6/](../../examples/v0.6/) — exit code **5** (`10 / 2` via `math.divide`).

---

## Not in v0.6

Struct params/returns, generics, enum params on `main`, qualified constructors (`math.Ok`).
