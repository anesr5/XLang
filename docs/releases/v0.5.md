# XLang v0.5 — Algebraic enums and match

**Status:** shipped in bootstrap compiler  
**RFCs:** [RFC-0031](../rfcs/RFC-0031-v0-5-roadmap-and-scope.md) through [RFC-0037](../rfcs/RFC-0037-v0-5-diagnostics.md)

---

## Summary

v0.5 adds **sum types** as a minimal enum system without generics:

- `enum` declarations with unit variants and single-field payloads (`i32`, `bool`)
- Variant constructors as calls: `None()`, `Some(42)`, `Ok(1)`, `Err(0)`
- `match` expressions with exhaustiveness checking
- LLVM lowering as tagged unions `{ i32 tag, i32 payload }` for stack locals
- Option/Result naming conventions (`OptionI32`, `ResultI32`)

---

## Language changes

| Feature | v0.4 | v0.5 |
|---------|------|------|
| `enum` declarations | — | yes |
| Variant constructors | — | `Some(42)`, `None()` |
| `match` expressions | — | yes, `=>` arms |
| Exhaustiveness | — | all variants or `_` |
| Enum in fn signatures | — | still rejected (ABI deferred) |

---

## LLVM lowering

- Enum locals: `%module.EnumName.tagged = type { i32, i32 }`
- Tag = variant index; payload widened to `i32` (`bool` zero-extended)
- `match` lowers to `switch` on tag with phi merge

## Not in v0.5

Generics, multi-field payloads, enum params/returns, `str` payloads at codegen, nested enums, `if let`, cross-module enum constructors.
