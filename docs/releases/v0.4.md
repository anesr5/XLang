# XLang v0.4 — Modules and multi-file compilation

**Status:** shipped in bootstrap compiler  
**RFCs:** [RFC-0023](../rfcs/RFC-0023-v0-4-roadmap-and-scope.md) through [RFC-0030](../rfcs/RFC-0030-v0-4-diagnostics.md)

---

## Summary

v0.4 turns the v0.1 syntax-only `module` / `import` declarations into a working **multi-file module system**:

- Deterministic module discovery from the entry file’s directory
- Transitive import loading with cycle and duplicate detection
- `pub` visibility for cross-module functions and struct types
- Qualified names: `math.add(...)`, `geom.Vec2`
- Per-module LLVM IR (`build/<module>.ll`) with mangled symbols (`@xlang.<module>.<fn>`)
- `clang` links all modules into one executable

Single-file programs without imports keep v0.1–v0.3 behavior (implicit `pub`, optional `module` name defaulting to `main`).

---

## Language changes

| Feature | v0.3 | v0.4 |
|---------|------|------|
| `import M` | parsed only | loads `M.x` or `M/main.x` |
| Duplicate import in one file | allowed | error |
| Circular imports | ignored | error with cycle path |
| `pub` on functions / structs | — | required for cross-module use |
| Qualified calls | — | `math.add(a, b)` |
| Qualified struct types | — | `geom.Vec2 p = { … };` |
| Multi-file `build` | `build/main.ll` | `build/*.ll` per module |

---

## Module layout

Project root = parent directory of the CLI entry file.

| Module `M` | Resolved path |
|------------|---------------|
| flat | `{root}/M.x` |
| nested | `{root}/M/main.x` |

The `module` declaration must match the path (`math.x` → `module math`).

---

## LLVM linking

- Entry module emits unmangled `@main`
- Other functions: `@xlang.<module>.<name>`
- Struct types: `%<module>.<StructName>`
- Cross-module calls use `declare` + `call` in the importer’s IR

## Not in v0.4

Package manager, re-exports, transitive import visibility, struct params/returns across modules (ABI), `use` imports, nested module paths.
