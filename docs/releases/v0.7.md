# XLang v0.7 - Struct values in function signatures

**Status:** shipped in bootstrap compiler  
**RFC:** [RFC-0043](../rfcs/RFC-0043-v0-7-struct-pass-return-abi.md)

---

## Summary

v0.7 lets declared struct types appear in function parameters and return types, including cross-module `pub struct` APIs.

---

## Language changes

| Feature | v0.6 | v0.7 |
|---------|------|------|
| Struct locals and field access | yes | yes |
| Struct fn params / returns | no | yes |
| Cross-module `pub struct` signatures | no | yes |
| Struct pass/return LLVM ABI | no | by value |

---

## LLVM ABI

Struct parameters and returns lower to named LLVM struct values passed by value:

```llvm
%geom.Point = type { i32, i32 }

define %geom.Point @xlang.geom.origin()
define i32 @xlang.geom.sum(%geom.Point %p)
```

Function parameters are copied into stack slots on entry so existing field access lowering continues to use addresses.

## Examples

See [examples/v0.7/](../../examples/v0.7/) - exit code **11** (`5 + 6` through `geom.Point`).

---

## Not in v0.7

Ownership, borrowing, nested structs, arrays in structs, generic structs, and bare expected-type struct literals such as `return { 1, 2 };`.
